# /interflect:review

Review Interflect promotion proposals. This command is the boundary between
retrospective lesson extraction and durable mutation.

## Review policy

For each proposal:

1. Verify the source handle and snippet support the claim.
2. Confirm the target substrate is correct:
   - memory for compact stable cross-session facts;
   - skill patch for reusable procedures/pitfalls;
   - repo doctrine for project-local canon;
   - Beads follow-up for work that should be tracked;
   - routing signal for Interspect/overlay evidence;
   - runtime-only for transient state that should not be promoted.
3. Accept, reject, or reclassify.
4. Apply only with an explicit operator-approved apply path.

## Guardrail

No automatic mutation has been applied by analysis. Memory, canon, skill, and
routing changes require explicit approval. Beads captures must still check for
near-duplicates before creation.
